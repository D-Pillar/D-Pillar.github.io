HTML and CSS Elements

Basic Foundation and set up of a HTML File

Name Description Placement Example
<!DOCTYPE html> A declaration that tells the browser what version of HTML the document is written in. First line of a HTML file. <!DOCTYPE html>
<html> Represents the root of the document, often reffered to as the root element. Second line of HTML file, with a closing tag as the last line in the file. <html> </html>
<head> Contains machine-readable information about the document, such as <title> and stylesheet. Child element of <html> <head> </head>
<link> Specifies relationships between the current document and an external resource. Can be placed in either the <head> or <body> elements, though the file is more readable and organised if it remains in the former. <link href="style.css" rel="stylesheet" />

Where 'href' means Hypertext Reference, and 'rel' means Relationship.
<title> Sets the title of the page, which is the title that appears in the browser tab when the page is loaded in. Placed in the <head> element. <title> </title>
<body> This contains all of the content displayed on a page. Placed after the closing tag of <head>, with a closing tag as the second last line in the file. <body> </body>

The Basic Elements Found in the <body> Element

Name Description Placement Example
<header> Not to be confused with <head>, <header> represents the introductory content of a page, typically a group of introductory or navigational aids. Child element of <body> <header>
  <h1>This is an example.</h1>
  <p>And another example.</p>
</header>
<nav> Represents a section of a page whose purpose is to provide navigational links, such as menus tables of contents, and indexes. Child element of <body>, and usually comes directly after the <header> element. <nav> </nav>
<h1> <h2> <h3>
<h4> <h5> <h6>		 Represents six different levels of section headings, with <h1> being the largest and every heading after shrinking in size. Placed in the <body> element, often as the parent element of <p> <h1> </h1>
<h2> </h2>
<h3> </h3>
<h4> </h4>
<h5> </h5>
<h6> </h6>
<main> Represents the dominant content of the document. The main content area consists of content directly related to or expands upon the central topic of the document. Can often be found as the child element of <header> <main> </main>
<article> Represents a self-contained composition in a document, page, application, or site. Most often found in forum posts, magazine or newspaper articles, blog entries and so forth. Found in the <body> element, often with a heading element as its child. <article> </article>
<aside> Represents a portion of the document whose content is only indirectly related to the documents main content. Frequently presented as sidebars or call-out boxes. Often found as a child element of <article> <aside>
  <p>This is how it would be placed.</p>
</aside>
<section> Represents a generic stand alone section of the document that doesn't have a more specific element to represent it. Found in the <body> when an element doesn't have a specific parent element present. <section> </section>
<div> Defines a division or section in an HTML document to establish a more coherent flow. The tag can be placed anywhere within the <body> element. <div> </div>
<footer> A footer typically contains information about the author of the page, copyright data, or links to related documents. Typically found at the bottom of the file, before </body> <footer> </footer>

The Basic Elements of HTML Text

Name Description Placement Example
<p> Represents a paragraph. The tag can be placed anywhere within the <body> element. <p> </p>

Though it is possible a closing tag isn't needed, depending on what element proceeds the content of <p>
<ol> Represents an ordered list of items, typically rendered as a numbered list. Placed in the <body> element when needing to make a numbered or ordered list, such as for a recipe. <ol>
  <li>This is an example.</li>
  <li>And another example.</li>
</ol>
<ul> Represents an unordered list of items, typically rendered as a bulleted list. Placed in the <body> element when needing to make a list, <ul>
  <li>This is an example.</li>
  <li>And another example.</li>
</ul>
<li> Represents items in a list. Can be either ordered or unordered, meaning numbered or displayed with bullet points, depending on the parent element. A child element of <ol> or <ul> <ol> or <ul>
  <li>This is an example.</li>
  <li>And another example.</li>
</ol> or </ul>
<hr> Represents a thematic break between paragraph-level elements. For example, a change of scene in a story, or a shift of topic within a section. Often placed between two <p> elements. <hr> </hr>

Though, a closing tag is not always necessary.
